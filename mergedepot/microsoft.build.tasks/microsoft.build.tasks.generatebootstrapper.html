    GenerateBootstrapper Class

    In this Article

    Implements the GenerateBootstrapper task. Use the GenerateBootstrapper element in your project file to create and execute this task. For usage and parameter information, see GenerateBootstrapper Task.

      Gets or sets whether a component runs with elevated permissions when it is installed on a target computer.

      public bool ApplicationRequiresElevation { get; set; }

      Property Value

      • Boolean

        Returns True if the installed component runs with elevated permissions; otherwise False.

      Remarks

      This property can be set in the project file, but it is not exposed in the MSBuild schema. In the project file's MsiBootstrapper section, set the RequiresElevation Property to True.

      A component runs with elevated permissions when it writes a registry key or is being installed to the GAC. For example, if you are installing assemblies to the GAC, you need to set this property to True.

    • ComponentsLocation

      Gets or sets a location for the bootstrapper to look for installation prerequisites to install.

      public string ComponentsLocation { get; set; }

      Property Value

      • String

        The location for the bootstrapper to look for installation prerequisites to install.

      Remarks

      This property can have the following values::

      • HomeSite: Indicates that the prerequisite is being hosted by the component vendor.

      • Relative: Indicates that the preqrequisite is at the same location of the application.

      • Absolute: Indicates that all components are to be found at a centralized URL. This value should be used in conjunction with the ComponentsUrl property.

        If ComponentsLocation is not specified, HomeSite is used by default.
