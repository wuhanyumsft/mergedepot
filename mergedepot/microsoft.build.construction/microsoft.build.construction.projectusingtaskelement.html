    ProjectUsingTaskElement Class

    In this Article

    Represents a UsingTask Element (MSBuild) in an MSBuild project. The UsingTask element is used both for inline tasks and precompiled tasks.

    Syntax

    Declaration

    [System.Diagnostics.DebuggerDisplay("TaskName={TaskName} AssemblyName={AssemblyName} AssemblyFile={AssemblyFile} Condition={Condition} Runtime={Runtime} Architecture={Architecture}")]
public class ProjectUsingTaskElement : Microsoft.Build.Construction.ProjectElementContainer

      Property Value

      • String

        Returns the value of the AssemblyFile attribute. Returns an empty string if the attribute is not present.

      Remarks

      The assembly may have a strong name and may be located in the GAC. Either the or attribute must be present, but not both.

    • AssemblyName

      Gets and sets the value of the AssemblyName attribute.

      public string AssemblyName { get; set; }

      Property Value

      • String

        Returns the value of the AssemblyName attribute. Returns an empty string if the attribute is not present.

      Remarks

      The file path to the assembly, which may be either a full path or a relative path. Relative paths are relative to the directory of the project file containing this UsingTask. Either the or attribute must be present, but not both.

      Gets the inner XML content of this inline task.

      public Microsoft.Build.Construction.ProjectUsingTaskBodyElement TaskBody { get; }

      Property Value

      • ProjectUsingTaskBodyElement

        Returns the inner XML content of the inline task. Returns null if no body exists.

      Remarks

      The inner XML content is the source code body of the inline task, and is passed directly to the task factory as a string.

    • TaskFactory

      Gets and sets the value of the TaskFactory attribute of this inline task.

      public string TaskFactory { get; set; }

      Property Value

      • String

        Returns the value of the TaskFactory attribute. Returns an empty string if the attribute is not present.

      Remarks

      Selects the task factory class name used to create instances of this task. TaskFactory is used together with either the AssemblyName or AssemblyFile attribute to locate the task factory.
