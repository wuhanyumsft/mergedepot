      Property Value

      • Nullable<Boolean>
        Nullable(Of Boolean)

      Remarks

      If false, the TargetDedicated property is required.

      If true, the AutoScaleFormula property is required. The pool automatically resizes according to the formula.

      The default value is false.

      AutoScaleFormula
      AutoScaleFormula

      Gets or sets a formula for the desired number of compute nodes in the pool.

      public string AutoScaleFormula { get; set; }Public Property AutoScaleFormula As String

      Property Value

      • String
        String

      Remarks

      For how to write autoscale formulas, see https://azure.microsoft.com/documentation/articles/batch-automatic-scaling/. This property is required if AutoScaleEnabled is set to true. It must be null if AutoScaleEnabled is false.

      The formula is checked for validity before the pool is created. If the formula is not valid, an exception is thrown when you try to commit the PoolSpecification.

      VirtualMachineSize
      VirtualMachineSize

      Gets or sets the size of the virtual machines in the pool. All virtual machines in a pool are the same size.

      public string VirtualMachineSize { get; set; }Public Property VirtualMachineSize As String

      Property Value

      • String
        String

      Remarks

      For information about available sizes of virtual machines for Cloud Services pools (pools created with a CloudServiceConfiguration), see https://azure.microsoft.com/documentation/articles/cloud-services-sizes-specs/. Batch supports all Cloud Services VM sizes except ExtraSmall.

      For information about available VM sizes for pools using images from the Virtual Machines Marketplace (pools created with a VirtualMachineConfiguration) see https://azure.microsoft.com/documentation/articles/virtual-machines-linux-sizes/ or https://azure.microsoft.com/documentation/articles/virtual-machines-windows-sizes/. Batch supports all Azure VM sizes except STANDARD_A0 and those with premium storage (for example STANDARD_GS, STANDARD_DS, and STANDARD_DSV2 series).
