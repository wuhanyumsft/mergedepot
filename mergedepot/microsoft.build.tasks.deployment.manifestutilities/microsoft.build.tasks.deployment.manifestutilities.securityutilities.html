    • ComputeZonePermissionSet(String,PermissionSet,String[])

      Generates a permission set by computing the zone default permission set, removing any excluded permissions, and adding any included permissions.

      public static System.Security.PermissionSet ComputeZonePermissionSet (string targetZone, System.Security.PermissionSet includedPermissionSet, string[] excludedPermissions);

      Parameters

      • targetZone
        String

        Specifies a zone default permission set, which is obtained from machine policy. Valid values are Internet, LocalIntranet, or Custom. If Custom is specified, the generated permission set is based only on the parameter.

      • includedPermissionSet
        PermissionSet

        A PermissionSet object containing the set of permissions to be explicitly included in the generated permission set. Permissions specified in this parameter will be included verbatim in the generated permission set, regardless of or parameters.

      • excludedPermissions
        String[]

        An array of permission identity strings to be excluded from the zone default permission set. Permission identity strings can be computed using the PermissionSetToIdentityList method. This parameter is ignored if is not Internet or LocalIntranet.

      Returns

      • PermissionSet

        Returns the generated permission set.

    • SignFile(X509Certificate2,Uri,String)

      Signs a ClickOnce manifest or PE file.

      public static void SignFile (System.Security.Cryptography.X509Certificates.X509Certificate2 cert, Uri timestampUrl, string path);

      Parameters

      • cert
        X509Certificate2

        The certificate to be used to sign the file.

      • timestampUrl
        Uri

        Indicates that the file is to be timestamped by the timestamp server at the specified HTTP address. This parameter can be NULL if you do not want a timestamp.

      • path
        String

        The name of the file to sign.

      Remarks

      This function can only sign a PE file if the X509Certificate2 parameter represents a certificate in the current user's personal certificate store.

      This overload of the method calls SignTool.exe and therefore requires the .NET Framework 2.0 to be installed.

    • SignFile(String,Uri,String)

      Signs a ClickOnce manifest or PE file.

      public static void SignFile (string certThumbprint, Uri timestampUrl, string path);

      Parameters

      • certThumbprint
        String

        Specifies the thumbprint, which is the SHA1 hash of the signing certificate, which is kept in your personal certificate store.

      • timestampUrl
        Uri

        Indicates that the file is to be timestamped by the timestamp server at the specified HTTP address. This parameter can be NULL if you do not want a timestamp.

      • path
        String

        The name of the file to sign.

      Remarks

      This overload of the method calls SignTool.exe and therefore requires the .NET Framework 2.0 to be installed.

    • SignFile(String,SecureString,Uri,String)

      Signs a ClickOnce manifest.

      public static void SignFile (string certPath, System.Security.SecureString certPassword, Uri timestampUrl, string path);

      Parameters

      • certPath
        String

        The certificate to be used to sign the file. The certificate must contain a private key. The KeyUsage extension is optional, but if it is present, DigitalSignature must be set to True. The EnhancedKeyUsage extension is also optional, but if it is present, the certificate must contain object identifier (OID) support.

      • certPassword
        SecureString

        The certificate password.

      • timestampUrl
        Uri

        Indicates that the file is to be timestamped by the timestamp server at the specified HTTP address. This parameter can be NULL if you do not want a timestamp.

      • path
        String

        The name of the file to sign.

      Remarks

      This function is only for signing a manifest, not a PE file.
